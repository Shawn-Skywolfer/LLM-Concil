# 贤者议会 (Council of Sages)

一个基于单文件 React 应用的 AI 多智能体辩论模拟器。用户可以输入复杂话题，系统会动态招募 3-5 位 AI 专家组成"贤者议会"，进行多轮辩论并最终达成共识。

## 功能特点

- **动态智能体招募** - 根据话题自动生成具有不同专业背景的 AI 专家角色
- **多轮辩论** - 智能体之间进行多轮观点交流和辩论
- **流式响应** - 实时显示 AI 回复过程
- **用户干预** - 可在辩论过程中插入自己的观点
- **双视图模式** - 支持看板式和列表式两种展示方式
- **会话管理** - 本地存储辩论历史，支持导出为 Markdown
- **云端备份** - 可选的 Google Drive 集成实现云端备份
- **深度思考支持** - 支持显示带有推理过程的模型输出

## 快速开始

### 方法 1：直接打开

直接用浏览器打开 `index-V8.HTML` 文件即可使用。

### 方法 2：本地服务器

```bash
# 使用 Python
python -m http.server 8000

# 使用 Node.js
npx serve .

# 然后访问 http://localhost:8000
```

## 配置说明

首次使用需要配置以下内容：

1. **API 设置** - 在设置中配置 OpenAI 兼容的 API 端点
   - 默认使用 OpenAI API
   - 支持自定义 Base URL（用于其他兼容的 API 服务）

2. **API Key** - 输入你的 API 密钥（安全地存储在本地浏览器中）

3. **模型选择** - 选择用于辩论的模型
   - 建议使用 GPT-5 或同等级别模型以获得最佳效果（推荐DeepSeek-V3.2和GLM-4.7）
   - 可使用我的邀请链接，注册AIHubMix，使用最新的大模型：
   https://aihubmix.com/?aff=G1zJ

4. **高级设置**（可选）
   - 调整辩论轮数
   - 自定义智能体数量（3-5 位）
   - 启用深度思考显示
   - 配置 Google Drive 备份

## 技术架构

- **前端框架**: React 18.2.0 (通过 ESM 模块加载)
- **样式**: Tailwind CSS
- **状态管理**: React Hooks (useState, useEffect, useRef)
- **存储**: LocalStorage + Google Drive API
- **API 集成**: OpenAI 兼容的 /chat/completions 端点，支持流式响应

# 一个让AI互相辩论的单文件应用：我做了个"贤者议会"

## 前言

最近看到 Andrej Karpathy 的一番话，让我印象深刻。

他说：**别再问AI"你怎么看了"——这是错误的提示词策略。**

Karpathy 的核心观点是：不要把大语言模型看作实体，而应视作**模拟器**。当你问"你怎么看"时，根本就没有"你"存在。LLM只是根据训练数据的统计规律，模拟一个被工程化塑造出来的"AI人格"。

他建议更有效的提问方式是：

> "什么样的**一群人**适合探索这个话题？**他们**会怎么说？"

这个观点击中了我。既然如此，为什么不直接做一个工具，让多个AI角色坐在一起，模拟一群专家的辩论呢？

于是，有了**贤者议会（Council of Sages）**。

## 它是什么？

一个实践 Karpathy 建议的工具：让多个AI角色互相辩论。

你输入一个话题，系统会自动招募3-5位不同背景的"AI专家"，然后让他们进行多轮辩论，最后由主持人总结各方观点并尝试达成共识。

**最特别的是：整个应用只有一个HTML文件，双击就能跑。**

## 为什么要模拟"一群人"？

### Karpathy 的洞察

Karpathy 指出，当我们问AI"你怎么看"时，LLM会采用一个隐含的"个性嵌入向量"（personality embedding vector），基于此进行模拟回复。

但这个"你"是被刻意设计、工程化并硬加上去的——通过SFT和RLHF训练出来的"理想助手"人格。

**它本质上仍然建立在一个Token模拟引擎之上，而不是一个具有涌现性的"心智"。**

### 更好的策略

Karpathy 建议的"模拟一群人"策略，能够：

- **激活不同的知识分布**：每个角色调用训练数据中不同领域的专业权重
- **突破单一视角的局限**：看到同一话题的多维度分析
- **获得更深入的理解**：观点的碰撞往往产生新的见解

这正是贤者议会做的事情。

## 项目如何实践这个理念

### 1. 动态角色生成

当你输入一个话题，系统不会问"AI你怎么看"，而是：
- 首先分析这个话题适合什么样的专家群体
- 动态生成3-5个不同背景的角色
- 每个角色都有独特的专业立场和表达风格

### 2. 多视角的碰撞

在辩论过程中，你不会得到一个"AI的观点"，而是：

- **深度学习理论家**可能从统计学角度分析
- **行业实践者**可能从落地经验出发
- **风险顾问**可能强调潜在问题
- **乐观主义者**可能着眼未来发展

这些角色之间的互动，正是 Karpathy 所说的"模拟多种视角"的具象化。

### 3. 可验证与不可验证领域

Karpathy 提到，在可验证领域（如编程），专家视角的模拟通常是准确的。而在不可验证领域（如观点意见），情况更有趣。

贤者议会尤其适合后者——当你面对一个复杂话题、需要多角度思考时，让不同立场的"虚拟专家"辩论，往往能帮你理清思路。

## 核心功能

## 使用场景示例

### 场景1：决策辅助

"35岁该不该离开大厂创业？"

与其问ChatGPT"你怎么看"，不如让这些角色辩论：

- **风险分析师**：列举创业失败率和财务风险
- **连续创业者**：强调时机窗口和长期收益
- **职业规划师**：分析个人发展阶段
- **家庭顾问**：考虑家庭责任和稳定性

### 场景2：学术探索

"AI是否会取代程序员？"

不同背景专家会从：
- 技术发展角度
- 就业市场角度
- 历史经验角度
- 伦理哲学角度

各自展开论述。

### 场景3：头脑风暴

产品创意、营销方案、文章选题...把问题扔进去，让AI们互相激发灵感。

## 技术实现

如果你对技术感兴趣，这里有一些特点：

- **零构建架构**：单HTML文件，React通过ESM加载，Tailwind CDN，Babel standalone转译
- **流式响应**：使用SSE接收OpenAI兼容API的实时输出
- **状态管理**：纯React Hooks，没有引入任何状态管理库
- **持久化**：localStorage + Google Drive API

代码开源在 GitHub，欢迎star和提issue。

## 如何使用

### 第一步：获取项目

```bash
git clone https://github.com/Shawn-Skywolfer/LLM-Concil
```

或者直接下载 `index-V8.HTML` 文件。

### 第二步：配置API

打开文件后点击设置，填入：
- API Key（OpenAI或兼容服务）
- Base URL（可选，支持第三方API）
- 选择模型（建议GPT-4级别）

### 第三步：开始辩论

输入你感兴趣的话题，点击开始，然后静静看AI们表演。

Karpathy 说："承认'没有你'，反而让我们能更自由地定义'你是谁'，从而榨出模型的最强性能。"

贤者议会就是对这个理念的实践——不追求一个虚假的"AI观点"，而是模拟一群有血有肉的专家，让观点在碰撞中产生价值。
